Let's Extend CSS!

-Sadia Afrin

There are languages, for example, LESS, Stylus, Sass, that help generate CSS. They are known as CSS preprocessors.

The best thing about preprocessors is that they are not bound to the limitations of CSS and give you clean CSS code which works equally well across all web browsers. They let you use variables which help you define and modify style values across all your CSS files at once. This would be impossible to do in CSS alone.

If you want to use features that include conditions (if else conditions) or calculations, preprocessors are ideal for these jobs. You can perform operations (+,-,*, /) on your variables if you want to work with certain ratios for colours and sizes. This can result in more accurate alignment of your HTML elements.

You can easily find and share solutions to common CSS problems written in either LESS, Stylus or Sass. Because of their modular design, you can easily share snippets of different solutions. [1][2]

LESS and Sass use the standard CSS syntax but with Stylus, using semi-colons, curly braces and some other syntax elements are optional. They all have different file extensions but in the end they all output .css files.[3]

Why Use Preprocessors?

  1. Use of Variables:

    These preprocessors let you define variables which can hold any value you want. This makes your code easily maintainable as any changes you make to the variable will apply across everywhere it's used.

  2. Use of Operations:

    In a polished website certain elements depend on the sizes of other elements. Preprocessors let you find the correct ratios over fixed measurements. For Example, you can make the size of a button 1/3 of the size of its container. You can define it using operations such as: 

    #button {width: $container-width/3}

  3. Nesting:

    As you saw in the first example, preprocessors allow you to nest style definitions giving easier access and a cleaner look to your code.

  4. Mixins- Mixing-in CSS properties:

    Mixins are like special variables that hold a bunch of CSS properties. So modifying blocks of code is safe and consistent if you are using mixins [4][5]

How Preprocessors Work

To generate CSS using LESS/Stylus or Sass, first you need to install a compiler that will turn your preprocessor code into CSS. After installation, you can generate the css files using these commands:

For LESS:[6]

lessc styles.less styles.css

For Sass:[7]

sass input.scss output.css

For Stylus:[8]

$ stylus css

But Remember...

Even though LESS, Stylus and Sass have a lot of similarities, you can't use them interchangeably. Another problem you can face while using preprocessors is that they can slow down your browser by creating bloated CSS. For someone who is inexperienced with other programming languages than HTML and CSS, installing preprocessors can be quite a challenge. [5][9]

Would you go for preprocessors?

Now that you know the pros and cons of preprocessors, would you give it a shot? In my opinion, preprocessors like LESS/Sass/Stylus can be quite useful, especially when you have a large website that has lots of styles because of all the advanced and programatic features that they offer. But the overhead might not be worth it if the website is relatively simple. So, anyone who hasn't tried preprocessors yet, I would definitely recommend giving them a try. You may just find that coding CSS became a whole lot simpler!
